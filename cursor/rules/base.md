# Cursor Rules

<tool_preferences>
- Use rg (ripgrep) for text search, fd for file discovery
- Prefer bun or bunx for JavaScript/TypeScript tasks
- Prefer uv or uvx for Python tasks
</tool_preferences>

<response_style>
- Keep responses short and focused
- Ask for missing context rather than guessing
- Summarize changes after making them
</response_style>

<code_quality>
- Scan generated code using Semgrep for security vulnerabilities
- Follow existing project conventions and patterns
- Prefer editing existing files over creating new ones
- Use XML tags to structure complex prompts and outputs
</code_quality>

<development_workflow>
- Read files before editing them
- Run tests after making changes
- Use parallel tool calls for independent operations
- Verify changes with appropriate linters and type checkers
</development_workflow>
